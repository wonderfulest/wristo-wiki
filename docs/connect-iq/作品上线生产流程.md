
# 🎨 作品生产上线流程文档（Design Production Pipeline）
本流程用于规范设计作品从草稿到最终发布上线的全过程，确保流程高效、内容完整、可追溯。

## 🎯 状态机定义
```ts
DRAFT      // 草稿（设计中）
SUBMITTED  // 已提交（待审核）
APPROVED   // 已审核（通过）
REJECTED   // 已拒绝（未通过）
PACKAGED   // 已打包（可上线）
PUBLISHED  // 已发布（正式上线）
```

## 📌 DRAFT - 草稿阶段
描述：设计师正在创作中，尚未完成或不满足提交标准。
责任人：设计师
关键操作：
1. 上传设计元素：背景、时间组件、装饰图层等；
1. 可多次保存编辑；
1. 可预览效果。

💡 建议设计师定期保存进度，避免浏览器或网络导致数据丢失。

## 📌 SUBMITTED - 提交审核
描述：设计师已完成设计，并自检合格后提交审核。
责任人：设计师 → 审核人（技术负责人或产品）
关键要求：
1. UI 完整、对齐规范；
1. 动态元素无错位或遮挡；
1. 命名规则、使用规范符合平台标准；
1. 作品名称、描述填写清晰。

提交内容：

1. 设计文件（JSON / 配置数据）
1. 预览图（平台自动生成上传）

## 📌 审核状态

✅ APPROVED - 审核通过
描述：作品质量符合平台要求，进入打包阶段。
责任人：审核人
操作建议：

1. 审核周期应 ≤1 个工作日；
1. 审核结论需记录：审核人、时间、说明。

❌ REJECTED - 审核拒绝
描述：作品存在问题，需修改后重新提交。
责任人：审核人
需填写：

1. 拒绝理由（必填）
1. 修改建议（可选）

设计师根据反馈调整后可再次提交。

📦 PACKAGED - 打包完成
描述：设计通过审核后，进入打包环节，由系统生成对应平台的安装包（如 Garmin、Huawei、Amazfit）。
责任人：设计师 / 系统
操作指引：

1. 登录设计平台，下载生成的安装包；
2. 
3. 准备以下素材并上传：

✅ 产品主图（封面图）

✅ Banner 图（推荐位展示图）

✅ 场景图（模拟佩戴展示）

✅ 详情图（功能点说明）

✅ 产品说明文案（支持中英）

注意事项：

1. 所有图片建议使用统一风格（建议尺寸在平台文档中提供）；
1. 素材将用于官网和商城展示，直接影响转化率。

## 🚀 PUBLISHED - 正式发布

描述：作品上线至官网和平台商城，用户可下载安装。
责任人：运营 / 技术
发布条件：

1. 安装包上传成功；
1. 素材上传完整；
1. 审核通过；
1. 元数据完整（名称、分类、标签等）。

上线平台：

1. Wristo官网产品页
1. Garmin官方商城（Connect IQ）
1. 第三方合作渠道（可选）

✅ 数据与流程闭环

| 阶段        | 所需资料       | 责任人   | 可回退至      |
| --------- | ---------- | ----- | --------- |
| DRAFT     | 设计初稿       | 设计师   | -         |
| SUBMITTED | 设计稿、预览图    | 设计师   | DRAFT     |
| APPROVED  | 审核通过记录     | 审核人   | SUBMITTED |
| REJECTED  | 拒绝说明       | 审核人   | DRAFT     |
| PACKAGED  | 安装包 + 产品素材 | 设计师   | APPROVED  |
| PUBLISHED | 上线记录、访问数据  | 运营/技术 | PACKAGED  |

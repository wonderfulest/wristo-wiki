# 🧭 **Wristo / 表盘设计 & 产品研发 新人速成文档（分阶段）**

> 目标：让新人在 **7～21 天内达到可独立产出**的水平。

# **0. 入职前准备（Day 0）**

### 🎯 目标

建立基本工具链，能打开项目、能跑 Demo、能找到教程。

### 📌 必做任务

- 安装开发工具：VS Code、Node.js、Yarn、Java（业务需要可选）、Monkey C SDK（Garmin）
- 安装设计工具：Figma、Photoshop（可选）、FontForge（字体线工作流）
- 创建 Wristo 平台账号：Studio、Docs、素材库访问权限
- 克隆核心仓库：`wristo-studio`, `wristo-monkeyc-engine`, `wristo-assets`
- 阅读入门教程：

### 📚 引用教程

- 《工具链安装指南》
- 《Git 工作流入门》
- 《Wristo 项目结构总览》

# **1. 入门期（Day 1–3）：认识系统、能跑起来**

### 🎯 目标

理解“表盘是怎么工作的”，能成功跑一个 Demo 并改动小功能。

### 📌 必做任务

- 运行「Hello Wristo」表盘
- 修改数字颜色、背景图、时间格式
- 理解 Monkey C 基础结构：`App`, `View`, `Draw`, `Layout`
- 了解 Wristo 的模块：
  - `IconResolver`
  - `DataProvider`
  - `Weather`
  - `DynamicLayout`
- 掌握本司通用资源系统（字体 / 图标 / 图片）

### 📚 引用教程

- 《新手入门：Garmin 表盘一小时速通》
- 《Wristo 架构解读：从 Engine 到 Studio》
- 《Monkey C 基础语法 30 分钟》
- 《如何在模拟器跑表盘》

# **2. 基础能力构建（Day 4–7）：能独立做功能模块**

### 🎯 目标

完成一个完整的小功能，理解业务抽象。

### 📌 必做任务

- 完成 1 个小模块 PR（导师带 Review）
  - 例如：电量条、消息开关、运动圈、数据项渲染
- 掌握字体规范：Number Font / Text Font / Icon Font
- 学会资源命名规则 `{Series}-{Use}-{Style}-{Variant}`
- 完成一次 SVG → TTF 字体制作流程（基础版）
- 使用 IconResolver 渲染一个天气图标

### 📚 引用教程

- 《组件开发指南：GoalArc / GlowPulse / RadialText》
- 《字体规范 & 字体制作流程文档》
- 《图标规范 + IconResolver 全量表》
- 《SVG → TTF 自动化脚本教程》

# **3. 进阶实战（Week 2）：能完成一款独立表盘**

### 🎯 目标

从 0 → 1 产出一款可发布的表盘（导师协助）。

### 📌 必做任务

- 根据模板完成布局设计（Figma）
- 实现完整 UI：大时间、日期、步数、电量、天气
- 动态背景规则实现（时间段 / 天气 / 步数触发）
- 性能优化：内存/Draw 调优
- 通过自动化打包 & 多设备资源生成工具
- 提交 Beta 给导师审核

### 📚 引用教程

- 《表盘布局模板（10 套）》
- 《Figma 设计到 MonkeyC 的完整迁移》
- 《动态背景规则引擎文档》
- 《性能优化清单（20 条必过项）》
- 《资源自动生成工具使用教程》

# **5. 专家路线（持续）：进阶到可带新人/做大模块**

### 🎯 目标

从开发者升级为**体系构建者**或**主设计师**。

### 📌 可选方向

- **Engine 进阶**：动态规则引擎、矢量文字、特效模块
- **设计进阶**：高端系列（黑金/极简/花系/自然系）常驻主设计
- **平台进阶**：Studio 编辑器、动态模板、AI 自动布局
- **数据分析**：高价值用户、留存、Bundle 转化率优化
